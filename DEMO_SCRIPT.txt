[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the workflow. First, every incoming email hits our Security Checkpoint node to check for PII, prompt injections, and size limits. If any hazard is found, it routes immediately to the Security Event exception page. Otherwise, clean text goes to the Inbox Orchestrator. The orchestrator delegates analysis to the Categorizer sub-agent—which checks the sender's VIP status via our MCP server—and the Responder sub-agent—which retrieves previous communication history to draft a matching response. The results are parsed by the Check Approval node. If the email is routine or urgent, it auto-approves and completes. But if it is classified as Sensitive, the workflow pauses at the Human Approval node to wait for a manual sign-off.
